# تحليل هندسي: إضافة ميزة "تحميل كلمات مخصصة" للعب أونلاين ومحلياً

## 🎯 ملخص الجدوى
الهندسة الحالية للعبة **ممتازة جداً ومؤهلة تماماً** لاستقبال هذه الميزة بأقل قدر من التعديلات. السبب الرئيسي لذلك هو أن "المضيف" (Host) يقوم بتوليد اللوحة، وينشر (Broadcast) الكلمات كنصوص `Strings` داخل الكائن `GameState` كـ JSON.
**النتيجة السارة:** العملاء (اللاعبون الآخرون) **لا يحتاجون** لامتلاك ملف الكلمات الأصلي عندهم، بل سيستقبلون الكلمات المخصصة تلقائياً عبر الشبكة (LAN أو Appwrite) ويتم عرضها ببساطة!

---

## 🔍 1. كيفية عمل النظام الحالي للكلمات
في الملف `lib/features/game_board/providers/game_provider.dart`:
```dart
static GameState _generateInitialState() {
  // الكود الحالي يسحب من الداتا الثابتة دائماً
  final shuffledWords = WordDatabase.arabicWords.toList()..shuffle(random);
  // ... ثم يقطع أول 25 كلمة لإنشاء الـ Cards
}
```

عندما يبدأ "المضيف" اللعبة، ويتم تغيير حالة اللوحة، يتم تحويل اللوحة إلى JSON:
- **في Appwrite:** يتم دفع الـ JSON كله كـ String إلى العمود `game_state`.
- **في الـ LAN:** يتم بث رسالة `SocketMessage` تحتوي على الـ JSON لجميع الهواتف المتصلة بالواي فاي.

بفضل هذه المعمارية الموحدة (Single Source of Truth) التي تعتمد على بث الـ `State`، فإن أي كلمة جديدة يضعها المضيف ستطير عبر الشبكة للجميع بدون تعديل أكواد الشبكة!

---

## 🛠️ 2. التعديلات المعمارية البسيطة المطلوبة (ما يجب عمله لاحقاً)

لإضافة الزر والميزة دون تخريب النظام المستقر، نحتاج 3 نقاط فقط:

### أ) حقن حالة الكلمات المخصصة في `game_provider.dart`
يتم تعديل دالتي `resetGame` و `_generateInitialState` لتقبلا متغيراً اختيارياً (Optional).
التعديل المعماري سيكون كالتالي:
```dart
// في GameNotifier
void resetGame({List<String>? customWords}) {
  state = _generateInitialState(customWords: customWords);
}

static GameState _generateInitialState({List<String>? customWords}) {
  // إذا المضيف أدخل كلمات، استخدمها. إذا لا، استخدم الأساسيات.
  final sourceWords = customWords ?? WordDatabase.arabicWords;
  final shuffledWords = sourceWords.toList()..shuffle(random);
  // تأمين النظام: يجب أن يحتوي المصدر على 25 كلمة على الأقل لبناء اللوحة.
  // ... باقي الكود يبقى كما هو.
}
```

### ب) شاشة الإعدادات أو اللوبي المتصل (`room_settings_screen` / `lobby_screen`)
1. **واجهة الزر (UI):** إضافة زر "استيراد كلمات مخصصة".
2. **مصادر التحميل:** 
   - **الخيار 1:** حقل نصي كبير يكتب/ينسخ المستخدم فيه الكلمات مفصولة بفاصلة أو سطر جديد.
   - **الخيار 2:** إمكانية رفع ملف `.txt`.
3. **طريقة الإرسال:** عند ضغط المضيف على "بدء اللعبة" (Start Game)، نقوم بتمرير هذه القائمة:
```dart
ref.read(gameProvider.notifier).resetGame(customWords: myCustomWordsList);
connection.startGame(); // ستبث اللوحة الجديدة كالعادة
```

### ج) أمان الشبكة والقيود (Appwrite & LAN)
1. **قيود اللوحة:** اللعبة تحتاج حصرياً إلى **25 كلمة** لتكوين كروت. 
   - *الشرط الهندسي:* إذا رفع المستخدم قائمة بها 20 كلمة فقط، يجب أن نظهر له رسالة خطأ (أو نكمل الباقي من الكلمات الـ Default لسد النقص).
2. **حجم البيانات عبر Appwrite:** حقل `game_state` في داتا بيز Appwrite هو نوعه string/json. كلمة من 10 حروف (بالبايت) * 25 كلمة = تقريباً 500 بايت ككلمات فقط، والـ JSON الخاص بالـ `WordCard` ككل صغير. لن يتم كسر قيود Appwrite لأن التعديل مجرد نصوص.
3. **حجم البيانات عبر LAN:** بروتوكول `Socket` المحلي ينقل حتى 64 كيلو بايت للباكيت براحة المعتادة. 3 كيلو بايت لا تشكل أي عائق.

---

## 🏆 3. الخلاصة للعب الفعلي
**لا تحتاج لتغيير كود الـ Multiplayer كلياً!**
كل ما عليك هندسياً هو:
1. بناء نافذة قراءة (Dialog) تأخذ نصوصاً من المضيف، تقسمها (Split)، وتنظفها من المسافات الفارغة.
2. التأكد أن عددها >= 25.
3. التمرير إلى محرك اللعبة (Game Engine) لتكوين `GameState` الجديد. 

محرك اللعبة سيتكفل بالباقي من مزامنة عالمية عبر Appwrite أو محلية عبر LAN دون أي جهد إضافي!
